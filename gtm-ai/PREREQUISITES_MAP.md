### Depend on ICP Definition

Run [ICP Definition & Refinement](skills/icp-definition-refinement/) first, then use its output as input to these skills:

- [ABM Program Playbook](skills/abm-program-playbook/) -- ABM targeting requires a crisp definition of your ideal accounts. Without ICP data, account selection becomes guesswork.
- [In-Market Account Takeover Campaign](skills/in-market-account-takeover-campaign/) -- Identifying in-market accounts starts with knowing what "ideal" looks like. ICP firmographics and buying signals define the filter.
- [Land & Expand Playbook](skills/land-expand-playbook/) -- Land-and-expand motions must be tuned to the segments you defined in your ICP. Expansion paths differ by segment.
- [Whitespace Analyzer](skills/whitespace-analyzer/) -- Whitespace analysis is only meaningful relative to your defined target market. ICP establishes the boundary of what "full coverage" looks like.
- [Expansion Opportunity Scorer](skills/expansion-opportunity-scorer/) -- Scoring expansion potential requires ICP firmographics and fit criteria to evaluate which existing accounts have the highest ceiling.
- [BDR Outbound Strategy](skills/bdr-outbound-strategy/) -- Outbound targeting, messaging, and sequencing all depend on knowing your ICP. Without it, BDRs spray instead of snipe.
- [Financial Services ABM](skills/financial-services-abm/) -- Vertical ABM programs are sharper when built on top of a well-defined ICP narrowed to financial services segments.
- [Enterprise Expansion Playbook](skills/enterprise-expansion-playbook/) -- Enterprise expansion strategies must align with your ICP's firmographic and behavioral profile.
- [Enterprise SaaS Pipeline Push](skills/enterprise-saas-pipeline-push/) -- Pipeline acceleration programs need to target the right accounts from the start.
- [Campaign Execution & BDR Alignment](skills/campaign-execution-bdr-alignment/) -- Aligning campaigns with BDR outreach requires shared ICP definitions so both motions target the same accounts.

**Why:** Your ICP definition establishes the "who" -- firmographics, technographics, pain points, and buying triggers. Every downstream skill that involves targeting, segmentation, or account selection produces significantly better output when it can reference a concrete ICP rather than relying on generic assumptions.

---

### Depend on Competitive Intelligence

Run [Competitive Intelligence](skills/competitive-intelligence/) or [Competitive Battlecard](skills/competitive-battlecard/) first, then feed the output to these skills:

- [In-Market Account Takeover Campaign](skills/in-market-account-takeover-campaign/) -- Takeover campaigns specifically target competitor customers, so you need to understand competitor strengths, weaknesses, pricing, and switching triggers.
- [Sales Enablement Playbook](skills/sales-enablement-playbook/) -- Enablement content must arm reps to handle competitive objections. Without competitive context, reps walk into deals unprepared.
- [Pitch Narrative](skills/pitch-narrative/) -- A compelling pitch implicitly or explicitly positions against alternatives. Competitive intelligence provides the contrast points that make differentiation concrete.
- [Sales Enablement Kit](skills/sales-enablement-kit/) -- Battlecards, one-pagers, and talk tracks all need competitive positioning data to be useful in live selling situations.

**Why:** Competitive intelligence provides the landscape context that transforms generic materials into sharp, differentiated ones. Without it, sales and marketing assets tend to focus on features rather than relative advantages -- which is how buyers actually evaluate options.

### Depend on Personas (Recommended, Not Required)

Run [Persona Brief](skills/persona-brief/) first, then use persona profiles as input to these skills. These are marked **recommended** rather than **required** -- the skills produce usable output without personas, but the output is more targeted and specific with them.

- [Campaign Brief Generator](skills/campaign-brief-generator/) -- Persona data sharpens the audience definition, messaging angle, and channel selection in each brief. (Recommended)
- [Email Nurture Sequence](skills/email-nurture-sequence/) -- Nurture sequences need persona-specific messaging, cadence, and content selection for each stage. (Recommended)
- [Content Brief](skills/content-brief/) -- Knowing who will read the content determines the angle, depth, format, and distribution channel. (Recommended)
- [BDR Outbound Strategy](skills/bdr-outbound-strategy/) -- Outbound messaging must resonate with the specific human reading the email. Persona-level detail makes sequences feel personal rather than templated. (Recommended)

**Why:** Personas define the "who" at the individual level -- their role, goals, objections, preferred channels, and language. Content and campaigns that reference a specific persona are more targeted and tend to convert at higher rates. However, these skills can still generate solid output by asking for audience details inline, which is why the dependency is recommended rather than required.

## Recommended Learning Paths

### Path 1: Build Your Foundation (Weeks 1-2)

These three skills create the strategic bedrock that every other skill benefits from. Start here.

1. [ICP Definition & Refinement](skills/icp-definition-refinement/) -- 30 min. Define your ideal customer profile with firmographics, technographics, pain points, and buying triggers.
2. [Persona Brief](skills/persona-brief/) -- 20 min per persona. Build 2-4 buyer personas with roles, goals, objections, and preferred channels. Run this skill once per persona.
3. [Competitive Intelligence](skills/competitive-intelligence/) -- 45 min. Map the competitive landscape, key differentiators, and positioning gaps.

**Output:** ICP definition, 2-4 persona profiles, and a competitive landscape overview. These three artifacts are referenced by 30+ downstream skills and form the foundation for all subsequent paths.

---

### Path 2: Plan Your GTM Strategy (Week 3)

With your foundation in place, build the strategic plan.

1. [Growth Marketing Roadmap](skills/growth-marketing-roadmap/) -- 1 hour. Map initiatives to quarters with owners and KPIs.
2. [Executive Business Plan](skills/executive-business-plan/) -- 45 min. Synthesize everything into a board-ready narrative with market sizing and financial projections.

**Input from Path 1:** ICP data shapes your growth roadmap. Competitive intelligence informs positioning. Personas guide messaging and channel priorities.

**Output:** A quarterly roadmap and executive plan that tie your GTM motion together.

---

### Path 3: Execute Campaigns (Weeks 4-12)

Now execute. These skills are designed to be run repeatedly throughout the quarter.

1. [Campaign Brief Generator](skills/campaign-brief-generator/) -- 3 min per campaign. Run 4-8 times per quarter to brief each campaign.
2. [Content Brief](skills/content-brief/) -- 5 min per piece. Generate briefs for blog posts, guides, landing pages, and other assets.
3. [Email Nurture Sequence](skills/email-nurture-sequence/) -- 15 min. Build multi-touch email sequences for each campaign.
4. [LinkedIn Ads Strategy & Execution](skills/linkedin-ads-strategy-execution/) -- 30 min. Plan LinkedIn campaigns with targeting, creative, and budget.

**Supporting skills to layer in as needed:**

## How to Read Prerequisites

### Simple Linear Dependency (A then B)

```
icp-definition-refinement --> abm-program-playbook
```

Run A first. Its output feeds directly into B.

### Complex Dependencies (A + B + C then D)

```
icp-definition-refinement ---\
                              +--> in-market-account-takeover-campaign
competitive-intelligence ----/
```

Run both A and B first (in any order), then combine their outputs when running the downstream skill.

### Optional Prerequisites (Recommended but Not Required)

```
persona-brief - - - -> campaign-brief-generator
              (recommended)
```

Dashed line means B works without A, but produces better results with A's output. Most dependencies in this library are "recommended" rather than "required." Every skill can run on its own -- prerequisites improve quality, they do not gate execution.

## FAQ

**Do I have to follow these paths?**

No. The paths are suggested sequences, not mandatory curricula. Every skill can be run independently. The paths simply represent the order that tends to produce the best cumulative output.

**Can I run skills out of order?**

Yes. Skills will prompt you for any context they need. If you have not run a prerequisite skill, you can provide the equivalent information manually or let the skill work with what you give it. Output quality may be lower, but the skill will still produce a useful result.

**How long does each path take?**

| Path | Estimated Time | Cadence |
|---|---|---|
| Path 1: Build Your Foundation | 2-3 hours (depends on number of personas) | One-time, refresh quarterly |
| Path 2: Plan Your GTM Strategy | 1.75 hours | Quarterly |
| Path 3: Execute Campaigns | 1-2 hours initial setup, then 30-60 min per campaign | Ongoing, weekly |
| Path 4: Enable Sales | 4-5 hours initial build | One-time, refresh per quarter or product release |
| Path 5: Measure & Optimize | 2.5-3 hours initial setup, then 10 min/week | Ongoing |
| Path 6: Scale & Expand | 3-4 hours as needed | Months 2-3+ |

**I want to run a specific skill. What should I run first?**

Look up the skill in the Dependency Graph section above. If it appears under one of the "Depend on..." sections, run the listed prerequisite(s) first. If it appears only in the "No Prerequisites" section, you are good to go.

For the fastest lookup:

| If you want to run... | Run this first |
|---|---|
| Any ABM or account-targeting skill | [ICP Definition & Refinement](skills/icp-definition-refinement/) |
| Any sales enablement skill | [Sales Playbook](skills/sales-playbook/) + [Competitive Battlecard](skills/competitive-battlecard/) |
| Any campaign execution skill | [Campaign Brief Generator](skills/campaign-brief-generator/) |
| Any persona-dependent content skill | [Persona Brief](skills/persona-brief/) |
| Any measurement or attribution skill | [UTM Taxonomy Generator](skills/utm-taxonomy-generator/) |
